API功能说明1.	为用户创建具体应用的实例提供RESTFUL API调用接口，接口名称暂定为http://ip:port/instance，用户通过http POST请求来创建与应用的实例，用户必须提供的参数为应用的ID，实例的名称，实例需要的CPU大小和内存大小，实例的规模，可选参数为GPU的个数，是否需要SSD固态硬盘。返回信息为实例创建成功或创建失败。具体需要完成的工作有：1.	调用kubernetes的RESFUL API，根据指定应用的模板创建Replication Conrtoller，模板中replica参数设置为实例的规模，docker容器的CPU和内存也根据参数设置，根据GPU和SSD可选参数设置资源调度标签。创建Kubernetes的Service，为用户提供应用的访问入口；2.	Kubernetes上的Replication Controller和Service创建成功后在数据库中记录下相关信息，具体包括实例的ID，实例的名称，应用的ID，CPU大小，内存大小，实例规模，GPU大小，，是否使用SSD，创建时间，Kubernetes上实例对应的Service的访问IP和端口。2.	为用户调整实例的规模提供RESTFUL API调用接口，接口名称暂定为http://ip:port/instance，用户通过http UPDATE请求来更新实例，用户必须提供的参数为实例的ID，新的实例的规模。返回信息为实例调整成功或失败。具体需要完成的工作有：1.	根据实例的ID从数据库中查找出实例的名称，调用kubernetes的RESFUL API，根据指定应用的模板更新Replication Conrtoller，模板中replica参数设置为新的实例的规模；2.	Kubernetes上的Replication Controller调整成功后在数据库中更新该实例的旧配置信息和使用时间，以及记录实例的新配置信息和创建时间。3.	查询用户指定的实例的信息提供RESTFUL API调用接口，接口名称暂定为http://ip:port/instance，用户通过http GET请求来查询实例，用户必须提供的参数为实例的ID。返回信息为实例的详细信息。具体需要完成的工作有：根据实例的ID从数据库中查找出实例的名称，应用的ID，应用的名称，CPU大小，内存大小，不同时间段的实例的规模和持续时间，GPU大小，是否使用SSD，创建时间，Kubernetes上实例对应的Service的访问IP和端口。4.	为用户删除其所创建的实例提供RESTFUL API调用接口，接口名称暂定为http://ip:port/instance，用户通过http DELETE请求来删除实例，用户必须提供的参数为实例的ID，返回信息为实例删除成功或失败。具体需要完成的工作有：1.	根据实例的ID从数据库中查找出实例的名称，调用kubernetes的RESFUL API，根据实例的名称删除kubernetes中对应的Service和Replication Controller；2.	Kubernetes上的Service和Replication Controller删除成功后在数据库中更新该实例的删除时间。
